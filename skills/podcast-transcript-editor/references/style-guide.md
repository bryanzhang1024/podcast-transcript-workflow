# 播客文字稿编辑风格指南

## 标点符号使用规范

### 逗号使用
- 在并列成分之间使用
- 分隔插入语或附加信息
- 在较长的句子中分隔不同的意群
- 但避免过度使用逗号造成句子破碎

### 句号使用
- 表示完整句子的结束
- 将过长的句子分割成更易读的短句
- 口语化表达中适当保留自然停顿

### 破折号使用
- 表示话题的突然转变
- 强调或者突出某个观点
- 分隔补充说明的内容

### 问号和感叹号
- 严格按照原文的语气使用
- 避免过度使用感叹号

## 段落划分原则

### 段落长度
- 一般2-3句话为最佳
- 不超过5句话的段落
- 避免过长的段落让读者感到疲劳

### 话题转换
- 每个段落应该只包含一个核心观点
- 话题转换时应该另起一段
- 保持段落之间的逻辑连贯性

## 发言者标注规范

### 标注格式
- 使用加粗格式：**主持人：**
- 每行只包含一位发言者的内容
- 发言者名称应该简洁明确

### 发言者识别
- 如果有明确姓名，使用真实姓名
- 无法确定姓名时，使用逻辑称谓：**主讲人：**、**嘉宾：**、**提问者：**
- 保持一致的标注方式

## 强调和格式规范

### 加粗使用
- 只对关键术语和重要观点使用
- 每个段落不超过2-3个加粗内容
- 避免过度使用造成视觉疲劳

### 标题使用
- 长篇内容使用标题结构
- 使用 Markdown 标题格式：## 二级标题、### 三级标题
- 标题应该简洁且概括段落内容

## 口语化处理

### 口语转书面语
- 保留自然的语气但去除冗余表达
- 修正口语化的语法错误
- 保持原文信息完整

### 重复内容处理
- 保留重要的重复强调
- 去除无意义的重复表达
- 保持内容的连贯性
